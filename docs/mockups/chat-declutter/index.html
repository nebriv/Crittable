Chat-declutter mockups

Four directions for handling chaos when multiple parallel issues + multiple participants are in the same chat. Same chaos scenario in each one (mid-incident ransomware, three parallel tracks, two SOC analysts on separate problems, two press injects, manager trying to follow along).

Try this in each mockup: use the Viewing as switcher in the top-right to flip between CISO / Legal / SOC Alex / etc. and watch where the @-mention amber highlight appears. The message data is identical across all four — only the chrome changes.

Iteration 4 · full app shell

Mockup E (Hybrid) now sits inside the actual Play.tsx app shell so the chat-declutter chrome is judged in context. Top status banner ("Your turn — CISO" / "Submitted, waiting on…"), three columns mirroring lg:grid-cols-[220px_1fr_280px]: left = roster with online dots and active-turn highlight + creator-only management buttons (Force-advance · Inject critical · Whisper to role · End session) + AI status; centre = filter row + chat + composer; right = three rail tabs (Artifacts · Action items · Timeline) plus an always-visible Notes & follow-ups panel below.

Iter-3 operator-feedback fixes baked in: (a) per-message track-chip dropped — the colored track-bar on the left already conveys track membership, and AI-categorized tracks aren't reliable enough to deserve a prominent header chip on every line; (b) the artifact strip across the top moved into a Timeline rail tab; (c) Critical rail tab dropped — injects are already loud inline and the Critical filter pill at the top covers the "scope to injects" need; (d) full app shell restored (top banner, left roster + management, NotesPanel below the rail) so the mockup reads like Play.tsx.

Open caveat — AI-categorization reliability: tracks are AI-categorized, not authoritative. Track filters degrade gracefully (clear all = falls back to All); the track-bar is a hint, not a primary nav. If categorization drift becomes a problem in production, a manual-tag override on each message is the next step.

Earlier focus-group ranking: CISO + Comms + IR put A or D in their top 2; Legal preferred C; SOC preferred C for parallel-track separation. Threads (B) lost unanimously. Legal's iter-1 asks (privilege markers, regulator clocks) are intentionally not in E — this is a tabletop-rehearsal tool, not a litigation-defense tool. The "mark for AAR" affordance is shelved in favour of PR #115's highlight-to-notepad flow (see issue #117).

E · Hybrid (synthesis)

ITER-2 CANDIDATE

A's filter-rail chassis with iteration-1 cybersecurity-team fixes folded in. New: pinned-artifacts strip across the top, multi-select track pills AND-combined with a quality filter, sticky minute-anchor timestamps, synthetic track-opened lifecycle rows, hidden-mentions banner, header export buttons (timeline.md / full-record.md), and three right-rail tabs (Artifacts · Action items · Critical).

Switch the viewer (top-right) between roles to watch the @-mention highlight follow you; click the export buttons to download a markdown AAR skeleton or full chronological record.

A · Filter chips + side rails

CONSENSUS PICK

Flat chat stays linear. Top filter chips for All / @Me / Critical / Decisions / Track. Right rail auto-pins Artifacts, Decisions, Critical injects from existing tool calls. Zero clicks for @me, ≤2 clicks for everything else. AAR transcript still reads top-to-bottom.

Pros: lightweight, no schema change, defensible record. Cons: still one stream during peak chaos.

B · Slack-style threads

FOCUS-GROUP REJECTED v1

Replies tucked into a "N replies" chip; click to expand right pane. Main channel quiets dramatically — but Legal/IR-Lead flagged it fragments the timeline they need for the AAR. Mocked up so we can confirm the rejection on actual UI, not in the abstract.

Pros: less scroll. Cons: hides timeline, fragments AAR record, "addressed to me" more painful to spot.

C · Multi-track lanes

EXPERIMENTAL

Three parallel column lanes (Containment / Disclosure / Comms / Lateral) plus a top "Main" lane for facilitator + injects. Each message lives in exactly one lane. Cross-lane @-mentions surface as amber dots on the foreign lane's header.

Pros: visual parallelism matches the actual work. Cons: 4 columns at once is a lot; doesn't survive mobile or 1080p well.

D · Tag tabs (lightweight)

SIMPLEST

One stream, every message tagged (#containment, #disclosure, …), top tab bar for filtering. Functionally a subset of A — no side rails. The bare minimum that addresses the cluttered-chat complaint without introducing new state machines.

Pros: cheapest to build, can ship in a day. Cons: no pinned artifacts/decisions, fewer affordances during chaos.
